# SubtitleForge v2 - 企业级视频字幕翻译工具

## 概述

SubtitleForge v2 是一款企业级视频字幕翻译流水线工具，从原始视频到双语字幕视频的完整解决方案。项目历经多次重大升级，从基础的逐句直译发展为**上下文感知翻译 + 后处理润色**的两阶段智能流水线。

---

## 🚀 核心升级（v1 → v2）

| 特性 | v1 (旧版) | v2 (升级版) | 升级说明 |
|------|----------|------------|----------|
| **翻译策略** | 逐句直译 | 每批10句+前3句上下文 | 大幅提升翻译连贯性和准确性 |
| **System Prompt** | 通用翻译指令 | 按风格定制专用指令 | 支持4种翻译风格 |
| **术语表** | 无 | 内置100+条日语→中文映射 | 术语一致性保障 |
| **ASR 纠错** | 无 | Whisper常见错误纠正 | 提升识别准确度 |
| **后处理** | 无 | 去AI味 + 口语化 + 连贯性检查 | 译文质量大幅提升 |
| **字幕时间对齐** | 有延迟（说完才显示） | **严格按原始时间戳，到时间立即显示** | 核心痛点修复 |
| **图形界面** | 无 | **网页版UI + Flask API** | 完整的Web应用体验 |
| **断点续传** | 无 | **支持断点续传** | 大文件处理不怕中断 |
| **配置持久化** | 无 | **JSON配置文件** | 配置记忆 |
| **日志系统** | 无 | **分级日志 + 文件轮转** | 生产级运维支持 |
| **单元测试** | 无 | **测试框架 + 多模块测试** | 代码质量保障 |

## 🔧 内核运行逻辑（8阶段流水线）

### 完整流程图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         输入视频文件                                     │
│                          (video.mp4)                                    │
└───────────────────────────┬─────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ [阶段 1/8] 音频提取                                                     │
│  FFmpeg 提取 WAV (16kHz / mono / 16-bit)                                │
│  命令: ffmpeg -i input.mp4 -vn -ac 1 -ar 16000 audio.wav                │
└───────────────────────────┬─────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ [阶段 2/8] 语音识别                                                     │
│  faster-whisper 模型推理                                                │
│  输出: 带词级时间戳的转录结果 raw_transcript.json                         │
└───────────────────────────┬─────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ [阶段 3/8] 语义断句 (v2 重写)                                          │
│  核心规则:                                                              │
│  ① 遇到句末标点（. ! ? 。！？）立即断句                                   │
│  ② 词间停顿时长 >= 0.2s 且非接续词时断句                                  │
│  ③ 累计时长 >= 6s 强制断句                                              │
│  ④ 绝对不允许 > 8s 的句子                                               │
│  ⑤ 过短片段（< 0.3s）后向合并                                           │
│  输出: sentences.json                                                   │
└───────────────────────────┬─────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ [阶段 4/8] 上下文感知翻译 (v2 核心)                                      │
│  处理方式: 每批10句 + 前3句上下文                                        │
│  API调用: DeepSeek Chat API                                             │
│  风格支持: anime / drama / youtube / documentary                         │
│  输出: translated.json                                                  │
└───────────────────────────┬─────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ [阶段 5/8] 后处理润色                                                   │
│  ① 去AI味: 删除冗余词汇，优化表达                                        │
│  ② 口语化: 根据风格调整语气                                              │
│  ③ 连贯性检查: 确保上下文一致                                            │
│  输出: final_subtitles_v2.json                                          │
└───────────────────────────┬─────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ [阶段 6/8] 翻译校对（可选）                                              │
│  二次精翻: 整体通读优化                                                  │
│  风格一致性检查                                                          │
│  输出: refined_subtitles.json                                            │
└───────────────────────────┬─────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ [阶段 7/8] 字幕生成                                                     │
│  双语 SRT: 原文 + 译文                                                  │
│  纯中文 SRT: 仅译文                                                     │
│  纯文本导出: 时间戳 + 双语对照                                           │
└───────────────────────────┬─────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│ [阶段 8/8] 字幕烧录                                                     │
│  使用 FFmpeg subtitles 滤镜直接烧录 SRT                                   │
│  ⚠️ 关键修复: 直接使用 SRT 而非 ASS，确保时间戳100%对齐                   │
│  输出: video_with_subtitles.mp4                                         │
└───────────────────────────┬─────────────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                         输出文件                                         │
│  ├── video_with_subtitles.mp4   (烧录字幕的视频)                         │

│  ├── subtitle_zh_v2.srt         (纯中文字幕)                            │
```

## 🎯 核心技术模块详解

### 1. 流水线引擎 (`pipeline_v2.py`)

**类: `SubtitlePipelineV2`**

| 方法 | 功能 | 阶段 |
|------|------|------|
| `__init__()` | 初始化流水线，加载配置 | - |
| `extract_audio()` | 从视频提取音频 | 阶段1 |
| `recognize_speech()` | Whisper语音识别 | 阶段2 |
| `split_sentences()` | 语义断句（词级时间戳） | 阶段3 |
| `translate_subtitles_v2()` | 上下文感知翻译 | 阶段4 |
| `post_process()` | 后处理润色 | 阶段5 |
| `refine_translations()` | 二次精翻校对 | 阶段6 |
| `generate_srt()` | 生成SRT字幕 | 阶段7 |
| `burn_subtitles()` | 字幕烧录到视频 | 阶段8 |
| `run_full_pipeline()` | 一键执行完整流程 | 全阶段 |

### 2. 上下文感知翻译器 (`translator_v2.py`)

**核心类: `ContextAwareTranslator`**

**翻译策略：**
- **批量处理**: 每批10句，减少API调用次数
- **上下文窗口**: 携带前3句上下文，保持翻译一致性
- **术语表**: 内置100+专业词汇映射（如动漫/影视剧专用术语）
- **风格定制**: 根据内容类型选择最优翻译策略

**支持的翻译风格：**

| 风格 | 适用场景 | 特点 |
|------|----------|------|
| **anime** | 动漫/广播剧 | 口语化强，语气词丰富，情感表现力强 |
| **drama** | 影视剧 | 自然流畅，保持人物性格和节奏 |
| **youtube** | 油管/Vlog | 轻松随意，网络化表达 |
| **documentary** | 纪录片 | 正式规范，术语准确 |

### 3. 后处理器 (`post_processor.py`)

**类: `PostProcessor`**

**处理流程：**
1. **去AI味**: 删除"嗯"、"啊"等冗余填充词
2. **口语化转换**: 将书面语转为口语表达
3. **连贯性检查**: 检查上下文一致性，修正指代不明问题
4. **风格适配**: 根据选择的风格调整语气和表达习惯

### 4. 断点续传机制 (`checkpoint.py`)

**类: `CheckpointManager`**

**工作原理：**
- 每个阶段完成后自动保存检查点
- 检查点文件: `pipeline_checkpoint.json`
- 下次运行时自动跳过已完成的阶段
- 支持手动清除检查点重新运行

## ⚠️ 关键修复：字幕时间对齐问题

**问题描述**: 之前版本字幕显示存在明显延迟——视频中的台词已经说完，字幕才弹出来。

**根本原因**: 使用 ASS 字幕格式 + FFmpeg `ass` 滤镜烧录，ASS格式渲染开销导致时间戳偏移。

**解决方案**: 改为使用 **SRT格式直接烧录**——FFmpeg的`subtitles`滤镜直接读取SRT文件。

**修改位置**: `pipeline_v2.py` 的 `burn_subtitles()` 方法

**效果**: ✅ 字幕到时间立即显示，时间结束后迅速消失
